File Structure
Everything you need, organized and ready to deploy

│
├── contracts/

│
├── scripts/
│   └── deploy.js                   ← Hardhat deployment script
│
├── test/
│   └── CertificateRegistry.test.js ← Contract unit tests
│
├── backend/
│   ├── main.py                     ← FastAPI server (all API logic)

├── .env.example                    ← Template (safe to commit)
├── .gitignore                      ← Excludes .env, node_modules
└── README.md                       ← Project documentation
ℹ️
How the parts connect
The smart contract lives on Polygon. The backend reads/writes to it via Web3.py and serves a REST API. The web dashboard and mobile app call that API. The ABI file (CertificateRegistry.abi.json) is auto-generated by the deploy script and placed in the backend folder automatically.

Project Setup
Copy files and install all dependencies

Create the project folder and copy all the downloaded files into it, maintaining the structure shown in Step 0. Then run:

bash — root setup
# 1. Navigate into the project root
cd certchain

# 2. Install Hardhat and blockchain tools
npm install

# 3. Copy the environment template
cp .env.example .env

# 4. Install backend Python dependencies
cd backend
pip3 install -r requirements.txt

# 5. Go back to root
cd ..
bash — mobile setup
# Install Expo CLI globally (once)
npm install -g expo-cli eas-cli

# Install mobile app dependencies
cd mobile
npm install
cd ..
✅
After this step you should have
node_modules/ in the root, mobile/node_modules/, and all Python packages installed. No errors should appear.
3
Compile the Smart Contract
Compile the Solidity contract and generate the ABI
bash — compile
# From the project root — compile the Solidity contract
npx hardhat compile

# Expected output:
# Compiled 1 Solidity file successfully
# Artifacts written to: artifacts/
ℹ️
What compilation produces
Hardhat compiles the Solidity contract and writes the ABI + bytecode to artifacts/contracts/CertificateRegistry.sol/CertificateRegistry.json. The deploy script will automatically copy the ABI to backend/CertificateRegistry.abi.json when you deploy.

Deploy Locally (Recommended First)
Run a local blockchain node and deploy — no wallet or testnet tokens needed
💡
Do this first — it's free and instant
Hardhat ships with a built-in Ethereum node. You can run the entire system locally without any real funds or internet connection. Perfect for development and testing.

Open two terminal windows. In terminal 1:

bash — Terminal 1: Start local blockchain node
# Start the local Hardhat node (keep this running)
npx hardhat node

# This prints 20 test wallets with 10,000 ETH each.
# Copy the first private key — you'll need it for .env
#
# Account #0: 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266
# Private Key: 0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
#
# Started HTTP server at: http://127.0.0.1:8545

In terminal 2, deploy the contract:

bash — Terminal 2: Deploy contract
# Deploy to the local Hardhat node
npm run deploy:local

# ═══════════════════════════════════════════
#   CertChain — Smart Contract Deployment
# ═══════════════════════════════════════════
# Deployer: 0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266

Deploy to Polygon Mumbai Testnet
When local testing is done — deploy to a real network with free testnet MATIC
⚠️
You need testnet MATIC first
Get free testnet MATIC from the Mumbai faucet: faucet.polygon.technology. Connect MetaMask to Mumbai (Chain ID 80001), paste your wallet address, and request tokens.
.env — Mumbai testnet config
RPC_URL=https://rpc-mumbai.maticvigil.com
CHAIN_ID=80001
PRIVATE_KEY=0xYourWalletPrivateKey   ← from MetaMask: Account Details → Export Private Key
MUMBAI_RPC_URL=https://rpc-mumbai.maticvigil.com
bash — deploy to Mumbai
npm run deploy:mumbai

# After deploy, update .env with the printed CONTRACT_ADDRESS
# The ABI is automatically copied to backend/CertificateRegistry.abi.json
6
Configure Google OAuth
Set up Google Sign-In for web and mobile
  • 1️⃣Go to console.cloud.google.com and create a new project (or select existing)
  • 2️⃣Navigate to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID
  • 3️⃣For Web: set Authorized redirect URIs to http://localhost:3000 and your production domain
  • 4️⃣For Android: create another OAuth client, select "Android", add your package name com.certchain.app and SHA-1 fingerprint
  • 5️⃣For iOS: create another OAuth client, select "iOS", add bundle ID com.certchain.app
  • 6️⃣Copy the Web Client ID — you need this in both .env and mobile/app.json
.env — add Google Client ID
GOOGLE_CLIENT_ID=123456789-abcdefg.apps.googleusercontent.com
ℹ️
Local testing without Google
For local development, you can skip real Google auth. The backend will still work — the Google token verification will fail gracefully and you can test all blockchain features without it.

Start the Backend Server
FastAPI + Web3.py — the API layer connecting frontend to blockchain
⚠️
Do Step 4 first
The backend needs backend/CertificateRegistry.abi.json to exist. That file is auto-generated by the deploy script. Make sure you've deployed the contract before starting the backend.
bash — start backend
# From the project root
npm run backend

# Or manually:
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# Expected output:
# INFO:     Started server process
# INFO:     Uvicorn running on http://0.0.0.0:8000
# INFO:     Application startup complete.

Verify the server is running and connected to the blockchain:

bash — health check
curl http://localhost:8000/health

# Expected response:
# {"status":"ok","chain_connected":true,"chain_id":31337,"block":1}

Interactive API docs available at:

Launch the Web Dashboard
Zero-dependency single-file HTML app

The web dashboard is a single index.html file with no build step required.

bash — serve the web app
# Option A: use the npm script (installs serve if needed)
npm run web

# Option B: use Python's built-in server
python3 -m http.server 3000

# Option C: just open the file directly in your browser
open index.html        # macOS
start index.html       # Windows
xdg-open index.html   # Linux

# Dashboard is at: http://localhost:3000
✅
Web dashboard works standalone
The web dashboard can run without the backend — all UI features are interactive. The blockchain calls will be simulated. Once the backend is running on port 8000, all live features activate automatically.

Run the Mobile App
React Native + Expo — preview on device or emulator
bash — start mobile dev server
cd mobile
npm run start

# Expo will print a QR code in the terminal.
# Scan it with the Expo Go app on your phone.
#
# Or press:
#   a → open Android emulator
#   i → open iOS simulator (macOS only)
#   w → open in web browser

Point the mobile app at your local backend. Edit mobile/App.tsx line:

mobile/App.tsx — update API base URL
// Change this line near the top of App.tsx:
const API_BASE = 'http://localhost:8000';     ← for iOS simulator
const API_BASE = 'http://10.0.2.2:8000';      ← for Android emulator
const API_BASE = 'http://YOUR_LAN_IP:8000';   ← for real device on same WiFi
# Find your LAN IP: ifconfig | grep "inet " (Mac/Linux) or ipconfig (Windows)

Build production APK/IPA:

bash — production builds
# Login to Expo (free account required)
eas login

# Configure project (run once)
eas build:configure

# Build for Android (creates .apk / .aab)
eas build --platform android

# Build for iOS (creates .ipa — requires Apple Developer account)
eas build --platform ios

# Build both at once
eas build --platform all

bash — manual API tests
# With the backend running, test the full flow:

# 1. Issue a certificate
curl -X POST http://localhost:8000/issue \
  -H "Content-Type: application/json" \
  -d '{
    "recipient_name": "Alex Johnson",
    "institution": "MIT",
    "course": "BSc Computer Science",
    "issue_date": "2024-11-27",
    "threshold": 1
  }'

# Copy the cert_hash from the response, then:

# 2. Verify the certificate
curl -X POST http://localhost:8000/verify \
  -H "Content-Type: application/json" \
  -d '{"cert_hash": "0xYourHashHere"}'

# 3. Fetch certificate details
curl http://localhost:8000/certificate/0xYourHashHere

Go Live — Production Checklist
Everything to check before deploying to mainnet
🚨
Never commit your private key or .env file
Your .env file is already in .gitignore. Use a secrets manager (AWS Secrets Manager, HashiCorp Vault, or environment variables on your server) in production. Never paste private keys in code.
  • ☐Deploy contract to Polygon mainnet: npm run deploy:polygon
  • ☐Update .env with mainnet RPC_URL, CHAIN_ID=137, and new CONTRACT_ADDRESS
  • ☐Grant ISSUER_ROLE to authorized wallets via Hardhat console or Polygonscan
  • ☐Set a strong random JWT_SECRET (use: openssl rand -hex 32)
  • ☐Deploy backend behind nginx with HTTPS (Let's Encrypt / Certbot)
  • ☐Update mobile/App.tsx API_BASE to your production HTTPS URL
  • ☐Update Google OAuth redirect URIs to include your production domain
  • ☐Set up IPFS node or configure Pinata API credentials
  • ☐Run npm test — all 12 tests must pass
  • ☐Build mobile app: cd mobile && eas build --platform all
  • ☐Submit to App Store and Google Play
